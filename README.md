# 2026电赛备赛规划

## 比赛简介
<img width="208" height="208" alt="image" src="https://github.com/user-attachments/assets/1b19d54a-e45b-491d-8597-cd7edbd7c763" />


电赛全名全国大学生电子设计竞赛，单数年一次竞赛分评省奖和国奖，双数年为邀请赛一次评奖，均在全国参赛学校中评选

## 时间表

- 一月份官网发布通知
- 五月底结束报名
- 七月份底公布器件清单
- 随后不久官网公布赛题
- 根据题目要求利用进行四天三夜设计一个作品并撰写设计报告
- 从早上八点到下午三点将作品封箱在各赛区指定地点进行评测，最后还要以队单位参加综合测评

## 赛题分类与分析
一共有仪器类、通信类	、测控类、电源类、信号处理、控制类、飞行器、高频及通信类这几种赛题，考虑到专业相关、个人兴趣以及上手难度，我们将控制类赛题作为首选，但是与此同时在可能没有控制题的情况下，我们在时间富余情况下，还会进行电源题以及测控题的练习

## 控制类备赛

### 平台-小车

小车搭建所需模块：
- 底盘：
  - 三轮：具有更好的转向能力，适用于直角转弯，21年题
  - 四轮：速度更快，24年题
- 系统板：
  - STM32: 赛题不一定允许使用，2020、2022年限制使用MSP
  - MSP430/432: 资料少，部分场景性能低，价格贵
- 电源模块：
  - DCDC开关电源模块
  - LDO线性稳压器
  - 接线：12v电源-DCDC-5v传感器-LDO-3.3v单片机和传感器
- 直流电机驱动模块：
  - 需要注意参数：输入电压、逻辑电压、驱动电流、最大功率
- 直流减速电机、编码器模块：
  - 一定要购买配有编码器的电机
- 舵机模块：
  - 若对控制精度和速度有要求，可以选择FOC无刷电机
- 传感器：
  - 灰度传感器：循线用，无MCU
  - 陀螺仪模块
- 摄像头：
  - MT9V034: C语言编程，上限高，便宜
  - openMV: 使用micropython，编程简单，性能差，价格贵
- 通信模块：
  - HC-05蓝牙模块

### 时间规划
备赛时间大概八个月，主要分为三个阶段
#### 第一阶段：基础学习
- 必学项
  - 数电模电基础知识
- 机械与硬件部分
  -  熟悉使用solidworks去设计赛题所需实践平台
  -  熟练运用嘉立创EDA以及AD完成电路设计，在学习过程中了解元器件选型以及布局
- 电机控制
  - 了解比赛中常用的电机
  - 掌握PID、MPC、LQR等控制算法
  - 熟悉一些常见的控制平台如STM32、TI控制芯片等等
- 视觉部分
  - 了解常见Linux系统开发板如树莓派、Jestson nano、地瓜派等
  - 学习ROS2、OpenCV、深度学习以及Yolo相关知识

备赛学习过程需要每人每周在issues中记录，相互监督，便于调整学习进度和后续时间安排

#### 第二阶段：模块专项训练
两到三个星期为一个周期，完成一个综合性项目，同时在项目中完成流程的固化“选题 -> 方案设计 -> 分工 -> 开发 -> 联调 -> 总结”
预计完成三个小项目：
- 坡道电动行驶小车
   - 机械与硬件：

     - 设计并制作轻量化且抓地力强的四轮小车底盘。

     - 集成陀螺仪，实时感知车身俯仰角。

     - 优化电源管理，确保电机在爬坡时有足够电压。

  - 软件与算法：

    - 实现基于增量式PID的直流电机速度闭环控制。

    - 实现基于PD算法的巡线控制。

    - 尝试简单的坡道补偿：根据陀螺仪数据，在爬坡时微调电机PID参数或设定速度。
  - 系统联调：
    
    - 调试坡道启停参数
    
    - 优化循线精度
- 智能送药小车
   - 机械与硬件：

     - 设计一个简易装卸机构

     - 开发 “主从车通信接口”（HC-05 蓝牙模块集成到 PCB）

     - 设计从车跟随机构（红外测距传感器安装支架）  

  - 软件与算法：

    - 编写有限状态机，清晰定义“寻路->识别药房->取药->寻路->识别病房->卸药”等状态及转换条件。

    - 实现基于摄像头的数字或颜色识别，用于识别药房和病房编号。

    - 实现双车间的简单通信协议（如：发送“我已取药，你可以出发”指令）。
  
  - 系统联调：
    
    - 调试装卸机构

    - 优化双车通信延迟
- 小车跟随行驶系统
  
   - 机械与硬件
  
      - 主车加装 OpenMV 摄像头（识别自身定位标识）
        
      - 从车增加陀螺仪模块（采集转向角度）
  
      - 优化 PCB 电源设计（分离电机与视觉模块供电，避免干扰）
        
   - 软件与算法
  
      - 主车：OpenMV 识别标识并上传位置信息（ESP8266 WiFi）
     
      - 从车：融合 “红外测距 + 陀螺仪数据”，实现弯道同步转向

      - 动态速度匹配：主车加速 / 减速时，从车 1 秒内同步响应
   - 系统联调
      
      - 调试弯道跟随精度
      
      - 解决视觉数据延迟问题
      
      - 测试不同速度下的跟随稳定性
#### 第三阶段：比赛模拟以及冲刺优化
1. 进行至少2次为期4天3夜的真题模拟。严格模拟比赛环境，独立完成从选题到报告提交的全过程。
2. 每次模拟后进行深度复盘，重点优化方案选择策略、时间分配和团队协作模式。
3. 整理常用电路、代码模板、文档框架，形成团队的“知识手册”。
## 资源需求
**1.指导技术方向以及进度调控**
需要和老师一起沟通项目技术方案以及备赛进度规划
**2.实验室与设备支持：**
一个固定的实验工位/桌面，用于存放物料、进行焊接和长期调试。

希望可以借用实验室的数字示波器、直流稳压电源、信号发生器、万用表等基础仪器。

我们将制定使用规范，自觉维护工位整洁与设备安全。

**3.物料以及经费预算：**
|类别|详细内容|
|---|---|
|结构与底盘|	3D打印耗材、激光切割板材、螺丝螺母等标准件|
|核心控制器|	STM32F4/F1核心板、MSP430练习板、OpenMV/MT9V034/k210摄像头|
|执行与传感|直流减速电机带编码器、舵机、陀螺仪、灰度传感器包|
|电路与电源|PCB打样与SMT费用、电机驱动模块、DCDC/LDO电源芯片|
|耗材与杂项|接线柱、导线、电池、接插件、焊接耗材等|

根据进度适时采购，并且过程中大量模块会复用
